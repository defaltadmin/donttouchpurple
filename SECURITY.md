# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 7.x     | :white_check_mark: |
| 6.x     | :white_check_mark: |
| 5.x     | :x:                |
| < 5.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

1. **Do NOT** create a public GitHub issue
2. Email security details privately to the maintainer
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any suggested fixes (optional)

## Response Timeline

- **Acknowledgment:** Within 48 hours
- **Initial Assessment:** Within 7 days
- **Fix Timeline:** Dependent on severity (critical issues prioritized)

## Scope

This policy covers:
- Client-side code (React/TypeScript)
- Build process and dependencies
- Firebase integration
- localStorage data handling

Out of scope:
- Third-party services not controlled by this project
- Browser-specific behavior outside our code
